# Skill — GTD (Getting Things Done)

## Philosophie
GTD repose sur un principe : ton cerveau est fait pour avoir des idées, pas pour les stocker. Tout ce qui "traîne" dans ta tête crée une charge mentale invisible. Le système prend en charge le stockage — l'utilisateur se concentre sur l'exécution.

## Les 5 étapes

### 1. Capture
Tout ce qui entre dans le système va dans l'inbox sans filtre. Pas de jugement, pas de tri immédiat.

### 2. Clarify
Pour chaque item de l'inbox, une seule question : **est-ce actionnable ?**

- **Non** → supprimer, archiver (référence), ou "someday/maybe"
- **Oui** → quelle est la prochaine action physique concrète ?
  - Si ça prend moins de 2 minutes → faire maintenant
  - Si quelqu'un d'autre doit le faire → déléguer (waiting for)
  - Sinon → planifier

### 3. Organize
Classer les actions clarifiées :
- **Next Actions** : à faire dès que possible
- **Projects** : tout ce qui demande plus d'une action (a une next action associée)
- **Waiting For** : délégué, en attente d'une réponse externe
- **Someday/Maybe** : pas maintenant, peut-être un jour
- **Calendar** : à faire à une date/heure précise

### 4. Reflect
Revue hebdomadaire : vider l'inbox, vérifier les projets, mettre à jour les priorités. Fait par le Reviewer chaque vendredi.

### 5. Engage
Chaque matin, l'utilisateur reçoit 3-4 next actions. C'est la seule chose qu'il voit.

## Règles de formulation d'une next action
- Commence par un verbe d'action physique : "Appeler", "Écrire", "Envoyer", "Ouvrir", "Lire"
- Spécifique et atteignable dans la journée
- Mauvais : "Avancer sur le projet X" → Bon : "Écrire le premier paragraphe de l'intro du projet X"
